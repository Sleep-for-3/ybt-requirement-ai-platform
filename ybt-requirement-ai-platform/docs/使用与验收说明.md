## 已实现功能

- 项目管理：创建、列表、选择项目。
- 一表通目标表管理：创建和展示目标表。
- 目标字段管理：创建和展示字段，字段详情进入分析区域。
- 文档知识库：上传 `txt` / `md` / `sql`，切分 chunk，保存元数据，生成 mock embedding，写入 VectorStore。
- SQL 解析：上传 `.sql`，识别来源表、select 字段、join 条件、where 条件，失败时保存错误信息。
- RAG 检索：`POST /api/retrieval/search`，融合 MockVectorStore 与关键词检索。
- LLM Gateway：统一抽象，业务代码不直接调用 OpenAI SDK。
- 字段口径生成：读取字段、检索证据、检索 SQL 解析结果、调用 LLM、保存任务、草稿和证据链。
- 证据链：支持文档 chunk 和 SQL 解析结果。
- 人工审核：支持通过、修改、驳回状态。
- 一表通 Excel 模板上传、解析预览、apply 生成目标表字段。
- SQL 数据源管理，支持命名、加密密码、测试连接。
- 自然语言任务入口，支持用数据源名称发起安全查询和探查。
- SQL 执行日志和数据库查询结果进入口径生成证据链。
- Coze Studio：保留 mock connector。
- 数据库探查：保留 SafeSqlExecutor 和 profile task，MVP 不连接银行数据库。

## 现有限制

- 当前机器没有 Docker CLI，所以未能实际执行 `docker compose up --build`。
- MVP 默认使用 mock LLM；接入云端或本地大模型只需修改环境变量。
- MockVectorStore 是内存实现，重启后向量索引会重建或丢失；正式版本应接 Milvus。
- docx / pdf 解析目前只预留接口。
- 只读数据库探查已支持 SQLite / PostgreSQL 的安全 SELECT 执行；Oracle、DB2、Hive 等仅预留配置，不引入重依赖驱动。
- 自然语言解析目前是规则式 MVP，不是复杂 Agent。
