# 一表通业务口径需求文档平台使用与验收说明

## 业务定位

本项目当前主线是“字段 × 产品场景的业务口径及技术溯源生产”：

```text
一表通固定字段结构
→ 历史业务口径及溯源 Excel 解析预览
→ 场景业务口径与贴源层技术溯源
→ 监管集市双层口径
→ Excel 交付工作簿和 Markdown 需求文档
```

SQL 解析、数据源安全查询、自然语言任务和数据库探查结果是辅助证据，不是最终交付物。

## 使用步骤

1. 创建项目。
2. 上传一表通 `.xlsx` 模板，并 apply 生成 `TargetTable` 和 `TargetField`。
3. 在 `/traceability-templates` 上传历史业务口径及溯源表，预览识别结果后再 apply。
4. 在 `/fields/{fieldId}/scenarios` 分场景维护业务口径和技术溯源。
5. 检索历史知识或生成候选来源，查看评分原因和证据摘要后人工选择。
6. AI 生成只写草稿；确认内容后点击“采用 AI 草稿”，再保存和确认。
7. 在 `/export` 导出真正的 `.xlsx` 交付工作簿。
8. 在“业务系统来源层”维护 `BusinessSystem`、`SourceTable`、`SourceField`。
4. 在“监管集市层”维护 `MartTable`、`MartField`，标记已有或建议新增。
5. 选择监管集市字段，创建“业务系统到监管集市口径”。
6. 选择一表通字段，创建“监管集市到一表通口径”。
7. 为每条口径绑定证据，例如人工备注、源字段、集市字段、模板、自然语言任务结果、SQL 执行日志等。
8. 点击 AI 草稿，生成业务规则描述。
9. 人工编辑 `final_content`，保存版本，审核通过或驳回。
15. 按项目、目标表或目标字段继续导出兼容的 Markdown 业务口径需求文档。

## 安全约束

- 主产物是业务口径需求文档，不是 SQL。
- 大模型不连接数据库。
- 大模型不自由执行 SQL。
- 所有查询必须通过 SafeSqlExecutor。
- 数据源密码加密保存。
- API 不返回 `password` 或 `encrypted_password`。
- SQL 解析和数据库探查结果只能作为证据引用。

## 当前限制

- 当前支持 Excel 和 Markdown，Word 导出未实现。
- `/legacy` 保留旧综合工作台；新路由已覆盖项目、模板、历史口径、字段场景、导出、数据源和任务入口。
- 数据源元数据自动采集未实现，源表字段和集市字段当前以人工维护为主。
- Mock LLM 便于本地验收，生产接入真实 OpenAI-compatible 或本地模型需配置环境变量。
- Milvus、Coze Studio、复杂 Agent 和本地大模型部署为后续扩展项。
