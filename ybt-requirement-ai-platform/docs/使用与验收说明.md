# 一表通业务口径需求文档平台使用与验收说明

## 业务定位

本项目当前主线是“字段 × 产品场景的业务口径及技术溯源生产”：

```text
一表通固定字段结构
→ 历史业务口径及溯源 Excel 解析预览
→ 场景业务口径与贴源层技术溯源
→ 监管集市双层口径
→ Excel 交付工作簿和 Markdown 需求文档
```

SQL 解析、数据源安全查询、自然语言任务和数据库探查结果是辅助证据，不是最终交付物。

## 使用步骤

1. 创建项目。
2. 上传一表通 `.xlsx` 模板，并 apply 生成 `TargetTable` 和 `TargetField`。
3. 在 `/traceability-templates` 上传历史业务口径及溯源表，预览识别结果后再 apply。
4. 在 `/fields/{fieldId}/scenarios` 分场景维护业务口径和技术溯源。
5. 在数据源目录页同步 SQLite/PostgreSQL/MySQL-compatible 元数据，或上传数据字典并在预览后确认 apply。
6. 在 `/catalog` 按数据源和 schema 分页浏览目录、懒加载字段，并按物理身份导入来源层或监管集市层。
7. 检索历史知识或生成候选来源，查看评分、推荐原因、字段元数据和探查历史。
8. 严格按“选择候选→安全探查→人工采用”执行；选择不会自动探查，探查不会自动采用。
9. AI 生成只写草稿；确认内容后点击“采用 AI 草稿”，再保存和确认。
10. 为业务口径、技术溯源和双层口径绑定脱敏证据；探查结果会生成 `column_profile` 证据。
11. 在“业务系统来源层”维护 `BusinessSystem`、`SourceTable`、`SourceField`。
12. 在“监管集市层”维护 `MartTable`、`MartField`，标记已有或建议新增。
13. 分别创建“业务系统到监管集市口径”和“监管集市到一表通口径”。
14. 人工编辑 `final_content`，保存版本，审核通过或驳回。
15. 在 `/export` 导出真实 `.xlsx`，或按项目、目标表、目标字段导出兼容的 Markdown 文档。
16. 在 `/knowledge/documents` 上传监管答疑、历史口径、制度、PDF、文本或 SQL 证据，并核对版本、解析状态和来源位置。
17. 在 `/knowledge/search` 验证混合检索分数和 citation，在 `/knowledge/ask` 验证无证据只输出“待确认”。
18. 在 `/evaluations` 运行人工确认的评测案例，查看 Recall@5、Recall@10、MRR、来源/字段命中率和引用覆盖率。

## 安全约束

- 主产物是业务口径需求文档，不是 SQL。
- 大模型不连接数据库。
- 大模型不自由执行 SQL。
- 所有查询必须通过 SafeSqlExecutor。
- 数据源密码加密保存。
- API 不返回 `password` 或 `encrypted_password`。
- SQL 解析和数据库探查结果只能作为证据引用。
- 数据源必须启用且标记只读，才能连接、同步、查询和探查。
- confidential/restricted 内容不得外发；允许外发的内容先脱敏，API 和 ModelCallLog 不保存明文密钥或敏感全文。
- citation 必须对应真实、启用且当前项目可见的知识单元。

## 当前限制

- 当前支持 Excel 和 Markdown，Word 导出未实现。
- `/legacy` 保留旧综合工作台；新路由已覆盖项目、模板、历史口径、字段场景、导出、数据源和任务入口。
- SQLite 元数据可直接同步；PostgreSQL/MySQL-compatible 已提供只读适配器并用 mock 验证，真实环境需配置只读连接和相应驱动。大规模同步本轮仍为同步任务执行。
- Mock LLM 便于本地验收，生产接入真实 OpenAI-compatible 或本地模型需配置环境变量。
- Milvus 适配器和可选 Docker profile 已实现；生产容量、备份、认证和高可用仍需环境专项验收。Coze Studio、复杂 Agent 和模型微调不在本轮范围。
